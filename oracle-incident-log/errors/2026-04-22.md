# Oracle Incident Log - errors

- [2026-04-22 16:54:51] Oracle Incident - errors
  - 질문 맥락: DB 6개가 이미 떠 있고 모두 `use_large_pages=only` 인 상태에서 새 DB를 추가로 open 하려는데 `ORA-27106` 발생
  - 에러 메시지: ORA-27106 system pages not available to allocate memory
  - 원인 가설: 새 인스턴스의 SGA 전체를 HugePages로 담아야 하는데 현재 서버에 예약된 HugePages(`vm.nr_hugepages`)가 기존 6개 DB와 ASM이 이미 사용 중인 SGA를 커버한 뒤 남는 양이 부족하다. 추가 가능성으로 새 DB가 `MEMORY_TARGET` 또는 `MEMORY_MAX_TARGET` 을 사용 중이면 `USE_LARGE_PAGES=ONLY` 와 충돌해 기동이 실패할 수 있다.
  - 권장 조치: alert log에서 HugePages 부족 관련 권장 page 수를 먼저 확인하고, 각 인스턴스의 `sga_max_size`, `sga_target`, `memory_target`, `memory_max_target`, `use_large_pages` 값을 점검한다. OS에서는 `/proc/meminfo` 의 `HugePages_Total`, `HugePages_Free`, `Hugepagesize` 를 확인해 새 DB SGA를 포함한 전체 요구량을 만족하는지 계산한다. 부족하면 `vm.nr_hugepages` 를 증설 후 재기동하거나, 새 DB SGA를 줄이거나, 임시 진단 목적이면 `use_large_pages=TRUE` 로 낮춰 mixed mode 기동 여부만 확인한다.
  - 검증 항목: `show parameter use_large_pages`, `show parameter sga`, `show parameter memory_target`, `show parameter memory_max_target` 결과 확인. OS에서 `grep Huge /proc/meminfo` 와 `ipcs -m` 또는 alert log 권장치 확인. 조치 후 새 DB open 시 `Large Pages Information` 섹션에서 전체 SGA가 large pages 로 할당되었는지 검증한다.
