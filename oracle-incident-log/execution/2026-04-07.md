# Oracle Incident Log - execution

- [2026-04-07 09:03:35] Oracle Incident - execution
  - 질문 맥락: OGG integrated 환경에서 초기 적재(expdp/impdp)와 CDC(Extract/Replicat)를 이어 붙일 때, Extract를 먼저 켜도 되는지와 Replicat 중복 방지 기준점(FLASHBACK_SCN, FILTER, PREPARECSN, ENABLE_INSTANTIATION_FILTERING, HANDLECOLLISIONS) 역할 정리 요청
  - 에러 메시지: 명시적 ORA 코드는 없으나, 초기 적재 후 Replicat 재적용 시 중복/누락 및 ORA-00001 가능성 우려
  - 원인 가설: Replicat가 없더라도 Extract는 trail에 변경분을 계속 적재하므로, 이후 Replicat를 기동할 때 full load 시점과 CDC 시작점을 같은 기준점으로 맞추지 않으면 impdp로 이미 적재한 행과 trail 재처리가 충돌할 수 있다. HANDLECOLLISIONS는 기준점 생성 기능이 아니라 보조 충돌 완화 옵션이다.
  - 권장 조치: 1) expdp 기준 SCN을 확보하고 Extract 시작점도 같은 SCN 또는 AT SCN으로 정렬 2) 수동 방식이면 Replicat에 FILTER(@GETENV('TRANSACTION','CSN') > 기준SCN) 적용 3) Oracle to Oracle initial load면 ADD TRANDATA ... PREPARECSN 후 expdp/impdp와 DBOPTIONS ENABLE_INSTANTIATION_FILTERING 조합 우선 검토 4) HANDLECOLLISIONS는 필요 시 한시적으로만 사용
  - 검증 항목: Replicat lag/logend/info로 backlog 소진 여부 확인, source/target row count 비교, FILTER 또는 instantiation filtering 제거 전 기준 SCN 이후 trail 소진 여부 확인
