# Oracle Incident Log - general


- [2026-04-08 09:12:50] Oracle Incident - general
  - 질문 맥락: OGG 19.24/19.29 환경에서 DMRMS(Source)와 SMRMS(Target, 분리된 Standby) 사이의 테이블 동기화 및 깨진 테이블 재동기화 준비 노트를 구조화하고, Data Guard 완료 상태 이후 OGG integrated 구성 흐름과 이슈를 정리함.
  - 에러 메시지: OGG-00706, OGG-10470, 그리고 두 OS 시간이 동일해 보여도 실제 동기화가 맞지 않아 후속 검증이 보류된 상태.
  - 원인 가설: SYS 소유 테이블은 Goldengate support mode가 INTERNAL로 나와 OGG 대상에 부적합했고, 권한 부족으로 supplemental log group 추가가 실패했다. 또한 환경 시간 불일치 또는 표시 포맷 차이로 최종 동기화 검증이 지연되었을 가능성이 있다.
  - 권장 조치: GGADMIN 사용자 테이블 기준으로 add trandata, integrated extract/replicat, credentialstore, checkpoint 정책을 재검증하고, DMRMS RAC 2-thread 및 SMRMS 단일 인스턴스 특성을 반영해 재동기화 절차와 시간 동기화 점검을 이어간다.
  - 검증 항목: GGSCI INFO ALL, SEND EXTRACT STATUS, DBA_GOLDENGATE_SUPPORT_MODE, 대상 행 변화 반영 여부, 시간 표시와 실제 OS/DB 시간 정합성을 함께 확인한다.
